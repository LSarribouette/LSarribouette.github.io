Rocket-Go en production

Tuto basé sur les conseils de Julien CALIENDO et le guide o2switch.

(cliquer sur les flèches pour plus de détails)

1. Hébergeur web

Une interface client :

  • on choisit un nom de domaine

Une interface d’administration technique : cPanel

  • on configure le nom de domaine

    ⚠️ ici, on met “nomdedomaine/public” comme racine du document parce que dans un projet Symfony, le fichier d’entrée index.php se trouve dans le dossier public

  • [optionnel] on crée un sous-domaine (autant qu’on veut)

2. Déploiement

  • Pour accéder à l’hébergement web o2switch, on se connecte via SSH.
    • Depuis le cPanel, on ajoute notre adresse IP dans la liste blanche SSH
    • Depuis un terminal, on se connecte en SSH en utilisant le même identifiant/mot de passe que cPanel : ssh identifiant-cPanel@nom-serveur.o2switch.net

      (afficher ce qu’il y a dans le répertoire : ls)

  • On ajoute le projet Symfony dans les fichiers de l’hébergeur.
    • on se place dans le répertoire du domaine ou du sous-domaine : cd nomdudomaine
    • on clone le repository souhaité depuis Github : git clone url .

      (créer un dossier du même nom que le projet : .)

    • si le clone échoue parce que le dossier n’est pas vide, on supprime tout ce qu’il contient : rm -Rf *
    • on installe tous les paquets nécessaires au projet : composer install

      (si le composer.lock bloque, on le supprime : rm composer.lock et on recommence)

    • on installe le paquet symfony/apache-pack pour pouvoir utiliser le serveur web Apache avec o2switch : symfony require symfony/apache-pack
    • au besoin, on change la APP_ENV pour passer en production : vi .env

      (grossièrement, x pour supprimer l’ancienne valeur dev >i pour insérer prod > echapechap pour arrêter l’insertion > majZmajZ pour sortir de vi)

C’est en ligne !!

  • Pour mettre à jour le site, on le pull depuis Github : git pull suivi d’un php bin/console cache:clear

    (on pense bien à se connecter en SSH à chaque fois qu’on veut interagir avec le serveur distant et à remodifier le .env au besoin)

3. Base de données

  • Les bases de données sont gérées via le cPanel, à l'aide des outils bases de données MySQL (ou PostgreSQL) de l'hébergement.
    • on crée la base de données souhaitée
    • on crée au moins un utilisateur pour la base de données
    • on lie l’utilisateur à la base de données
    • on donne des droits à cet utilisateur sur cette base de données

      (ici tous les droits, mais attention)

  • Depuis un terminal, on met à jour les informations de connection à la base de données.
    • on crée un fichier .env.local (jamais sur git) dans lequel on ajoute la DATABASE_URL : vi .env.local

      (grossièrement, i pour insérer DATABASE_URL="mysql://NOM_UTILISATEUR:MOT_DE_PASSE@127.0.0.1:3306/db_name?serverVersion=mariadb-10.3.28" > echapechap pour arrêter l’insertion > majZmajZ pour sortir de vi)

    • on met à jour la base selon la structure du projet : php bin/console doctrine:schema:update --force

      (oui c’est mieux les migrations)

  • Depuis PhpStorm, on ajoute des données dans la base.
    • on ajoute le .env.local avec la même DATABASE_URL
    • on se connecte normalement via l’onglet Database

      ⚠️ le nom de la base et le nom de l’utilisateur commencent par le login de cPanel : login_mabasededonnees et login_monutilisateur

    • on insert les données comme d’habitude

La base de données de production est prête et remplie !

4. Mettre à jour en production

  • On merge dans la branche main.
  • On se connecte en ssh.
  • On git pull via le terminal.
  • On se connecte à la base de données avec vi, en créant un fichier .env.local (jamais sur git) dans lequel on ajoute la DATABASE_URL : vi .env.local

    (grossièrement, i pour insérer DATABASE_URL="mysql://NOM_UTILISATEUR:MOT_DE_PASSE@127.0.0.1:3306/db_name?serverVersion=mariadb-10.3.28" > echapechap pour arrêter l’insertion > majZmajZ pour sortir de vi)
